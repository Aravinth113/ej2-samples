The TreeView component loads the data through the dataSource property, where the data can be either local data or remote data. In case of remote data, the data can be load from any remote services though our DataManager.

The DataManager will act as an interface between the service endpoint and the TreeView, which will require the below minimal information to interact with service endpoint.

  • DataManager->url - Defines the service endpoint to fetch data.
  • DataManager->adaptor - Defines the adaptor option. By default, ODataAdaptor is used for remote binding.

In this demo, the TreeView was bound with the dataSource from Northwind remote service by using the DataManager instance.

For more information, you can refer the Data Binding section from the documentation.
